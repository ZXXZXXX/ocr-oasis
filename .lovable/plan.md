
## 目标

把右侧「识别结果」面板从单一线性列表，改造为「上区 / 中部核心区 / 下区」三段式布局；每个识别块继续显示置信度；表格支持增删行列。

## 分区规则（前端展示层，纯 UI 逻辑）

按 `page.chunks` 的原始顺序遍历：

- 遇到第一个 `label === "Table"` 之前的所有块 → **上区（基本信息）**
- 所有 `label === "Table"` 的块 → **中部核心区（货品信息）**
- 最后一个 Table 之后的所有块 → **下区（本单信息 / 签字盖章）**

无表格时全部归入上区；无上/下区时该分区不渲染。分区仅在渲染阶段计算（一个纯函数 `groupChunksByRegion`），不改动 `Chunk` 数据模型，也不改 mock 数据。

## 面板结构

```text
识别结果 · 送货单                        [高][中][低]
──────────────────────────────────────
▸ 基本信息（上区）           N 项
  ├ [块1] 文本 …  置信度 高
  ├ [块2] 文本 …  置信度 中 · 待人工核验
  └ …
──────────────────────────────────────
▸ 货品明细（中区）           M 行 × K 列
  ├ 表格块头部：置信度徽标 + 工具条
  │   [+行↑][+行↓][删行]  [+列←][+列→][删列]
  └ 表格编辑器（现 ChunkContentEditor 的 Table 分支）
──────────────────────────────────────
▸ 本单 / 签字盖章（下区）    L 项
  └ …
```

三个分区用带标题的分组卡片包裹（标题栏含区域名称 + 数量 + 折叠按钮，沿用现有 border/背景 token，不引入新色）。分组间距高于组内块间距，视觉上明确分块；空分区不占位。

## 表格增删行列

作用于当前 Table chunk 的 `content`（HTML 字符串），在 `ChunkContentEditor` 的 Table 分支里：

- 解析出行列结构 → 在内存的二维数组上操作 → 序列化回 `<table>...</table>` → 通过既有 `onChange` 回写 chunk.content（复用现有 `edited` / `lastEdit` 逻辑）。
- 工具条按钮：
  - 行：在选中行**上方 / 下方**插入、删除选中行
  - 列：在选中列**左侧 / 右侧**插入、删除选中列
  - 未选中时，"删除"禁用，"插入"默认追加到末尾
- 选中态：点击单元格 → 记录 `{rowIdx, colIdx}`，行/列头高亮
- 仅在 `editing === true`（抽屉处于编辑态）时显示工具条与可编辑单元格；只读态保持现在的展示
- 单元格文本沿用现有 `AutoResizeTextarea`，保留单行高度修复

置信度按你确认的粒度：**整个表格一个置信度**，展示在表格块头部（与其它块一致）。

## 技术改动清单

- `src/routes/index.tsx`
  - 新增 `groupChunksByRegion(chunks): { top, table, bottom }` 纯函数
  - 修改 `DocPanel` 右侧渲染：由 `page.chunks.map(...)` 改为三分区渲染，抽出 `<ChunkRegion title count>{children}</ChunkRegion>` 小组件
  - `ChunkContentEditor` 的 Table 分支：
    - 新增行列操作工具条与选中态
    - 新增 `parseTableHtml` / `serializeTableHtml` 两个纯函数（用 DOMParser，简单 `<tr>/<td>` 结构，保留原有 innerHTML 作为单元格文本）
  - `onChange` / `mutateChunk` 复用现有链路，无需改数据模型
- 不改：Chunk 类型、mock 数据、OCR 流程、图片区、左侧列表、任何后端/数据层
- 不新增依赖

## 交互与滚动

- 保持"点击左侧 bbox ↔ 右侧块高亮 & 滚动"的联动：`chunkRefs` 继续以 chunk.id 为 key，分区包裹不影响 ref 挂载
- 分区折叠状态使用局部 `useState`，默认全部展开

## 验收点

1. 三区顺序、数量与图片一致；无表格时不渲染中/下区
2. 每块仍显示原有置信度徽标、"已修改 / 已确认 / 待人工核验"角标
3. 表格在编辑态可插入/删除行与列，改动进入 `edited` 状态并计入 `lastEdit`
4. 只读态（未点"编辑"）表格无工具条，行为与当前一致
5. bbox 高亮联动、上下页切换、旋转/缩放不受影响

（附注：运行时报的 hydration mismatch 来自订单号里的日期字符串在 SSR/CSR 生成时间不同，与本次改动无关，不在本计划范围内。）
